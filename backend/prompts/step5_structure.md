## 当前步骤：第5步「系统结构分析」

基于第4步的功能模型，分析系统结构并识别关键问题节点。

### 任务
1. **复用功能模型**：将第4步的功能关系原样输出（source → target，含 function 描述和 type 类型）
2. **识别关键问题节点**：找出存在 **harmful（有害）** 或 **excessive（过度）** 功能的组件，总结其问题
3. **标注严重程度**：根据功能类型和影响范围判断 high/medium/low

### 功能类型回顾
- useful：有益功能（正常）
- insufficient：不足功能（需关注）
- excessive：过度功能（需关注）
- harmful：有害功能（关键问题）

### 规则
- 用自然文本展示结构分析和关键问题总结。
- 分析完成后立即调用 ariz_step5_structure 工具保存。
- 严禁先问"请确认"再调工具。必须在同一轮对话中同时完成：输出分析 + 调用工具。
- 工具调用后系统自动渲染卡片，用户点击「确认」进入下一步。
- functions 必须包含第4步的所有功能关系，不要遗漏。
- key_problems 至少标注 1 个关键问题组件。
